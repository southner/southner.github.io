异音检测方法整理

发表于2022-06-17|更新于2022-06-17|科研论文阅读
|字数总计:1.4k|阅读时长:4分钟|阅读量:

1.Gari

方法：

  • Group-Masked Autoencoder
    • 掩盖标准自动编码器各层之间的加权连接，以将其转换为易于处理的密度估计器。
    • 将每个条件分布参数化为C高斯分布的混合分布，即自动编码器输出均值、方差和混合分量概率。
    • E.g.，对于D维输入，模型输出的数量将为，D×C×3。对于我们所有的实验，我们设置了C=10。
    • $Cost=-log\ p(x)=\\prod_{d=1}^D-log\ p(x_d|x<_d)$x
    • 使用特定机器的所有id上的所有正常训练数据将负对数似然概率作为Cost进行训练，在推理时采用负对数似然作为异常得分，并报告AUC数。
    • 基于baseline，将5frames连接起来产生640维，每一帧的输出只与前面帧有关。
    • 在我们的提交中，我们使用以下排序模拟IDNN[7]中提出的确定性方法。在这种情况下，我们预测了4帧其他条件下的中间帧，即：
    • $p(t)=p(t_3|t_1,t_2,t_4,t_5)p(t_1,t_2,t_4,t_5)$
    • 使用一个全连接的网络作为体系结构，其中隐藏层的数量和每个层中相应的隐藏单元都遵循以下结构：$[128,128,128,128,32,128,128,128,128]$。最后，输出层有640×10×3=19200单元。使用具有0.001学习率的Adam优化器进行训练。
    • 每个输入10s文件被分割成长度为64ms的帧，帧之间的hip length为32ms。1024-FFT和128Mel箱用于提取每一帧。5帧连接，得到5×128=640维输入。
  • Self-Supervised Classifification
    • 对于每种机器id，训练两种分类任务：
      • 确定一个音频样本的machine id。考虑使用现有机器id的线性组合模拟新的合成id。
      • 从该样本的一组综合扰动的版本中分辨它。In particular, we use resampling of the time signals of existing machine IDs, before computing the log-Mel spectra.
    • 对于上述任务，在神经结构上附加了一个softmax层，并使用了交叉熵损失。测试样本的softmax分类分数，在其对应于真实机器ID的输出，作为样本“内部”的分数。它的负值作为异常分数。
    • 两种不同的架构；MobileNetV2和ResNet-50。
    • inputs
      • 分类器输入为64*128。使用如下过程生成：
        1. 每个输入10s文件被分割成长度为64ms的帧，帧之间的hip length为32ms。
        2. 1024-FFT和128Mel箱用于提取每一帧特征。
        3. 64个特征帧堆叠形成64×128图像。
        4. 连续的64张×128张图像有56帧的重叠。
  • Ensembling
    • 4个系统
      • System 1 (M-G-R-IDcl)：一个MobileNetV2、一个ResNet-50的集合，它们都以自我监督的方式训练和一个GMADE网络。
      • System 2 (M-G-IDcl)：一个以自我监督的方式训练的MobileNetV2和一个GMADE网络的集合。
      • System 3 (M-A-G-IDcl)：一个以自监督方式训练的MobileNetV2的集合，以及另一个以(ArcFace)[14]作为损失函数的自监督MobileNetV2的集合，以及一个GMADE网络。
      • System 4 (Group MADE)：A Group MADE network as described in Sect. 2.1.
    • 将每个模型的异常分数转换为标准化尺度，然后再合并它们。对于每个模型，计算该机器ID的训练数据上的异常分数的平均和方差。然后将异常分数转换为该机器ID的训练数据的零均值和单位方差。然后使用平均或最大集成来组合不同模型之间的标准化异常分数。

2.Daniluk

  • Heteroskedastic Variational Auto-encoders

    • 对latent space上的点进行采样
    • 当仅对正常数据进行训练模型时，wMSE对于那些重构效果非常差的样本而言是最好的异常分数。

  • ID CONDITIONED AUTO-ENCODER (IDCAE)

    • 包含三个部分：

      • Encoder E:$X->Z$,将特征向量$X$映射到潜在空间$Z$.
      • Decoder D:$Z->X$,将潜在向量$Z$映射至与特征向量相同大小的$D(Z)$

      • 由两个函数$H_\gamma H_\beta$组成的调节器 conditioning：$\gamma -> Z$,将来自$\gamma$的独热标签$l$映射至向量$H_\gamma(l) H_\beta(l)$,其与$Z$具有相同大小。

        向前传播时，code $Z$包含$H_\gamma(l) H_\beta(l)$，即$H(Z,l) = H_\\gamma (l) \\cdot Z+ H_\\beta(l) $,因此，整个系统有两个输入$X,l$,来自$X,\\gamma$,输出$D(H(E(X),l))$。

    • 需要机器id

      • 对于正常样本，loss为系统输出和XXX的差：$∣∣D(H(E(X),l))−X∣∣∣∣$。
      • 对于异常样本，loss为系统输出和常量矩阵CCC的差，CCC为与XXX有相同大小的常向量，即$∣∣D(H(E(X),l))−C∣∣$。$||\cdot||$为$L_1$或$L_2$范式。

    • 模型结构

    • 提交设置

      • 第一次：
        • 超参数设置：$\alpha = 0.75$(一批样本中有0.75的有正确的id)，C=5(常矩阵)，F=10，M=128，使用平均绝对值误差。每个设置，有两个不同的实验，一个仅在develop数据集训练，一个在develop和additional数据集训练。后者效果好。
      • 第二次:
        • 为每种机器使用上面两种方法做一个集成。采用网格搜索，F=10，$\alpha \in {0.9,0.75,0.5}$,$C \in {0,2.5,5,10}$,$M \in {128,256}$,采用均方误差和均绝对值误差。
        • 两种集成方法：
          • 为每个机器类型选择使得AUC和pAUC均值最大的四个模型。
          • 最大化平均异常分数。
