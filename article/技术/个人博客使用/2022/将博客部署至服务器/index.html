将博客部署至服务器

在github.io配置hexo后，想要把hexo部署在服务器上，需要先在服务器上安装一些包，但yum不可用

使用apt-get install yum安装yum报错

解决方法是换源

安装yum

  1. 修改镜像下载地址

    先备份文件/etc/apt/sources.list

  3. 添加镜像地址

  4. vim一些操作方法

  5. 更新镜像

  6. 更新软件

  7. 更新后重新安装yum 提示有一些依赖的库没有安装

  1. 安装依赖库

    安装成功

服务器端操作

搭建远程Git库

1.安装git和nginx
2.添加一个git用户

#项目目录如下
-home
-git
-projects
-blog #项目存在的真实目录
-repos
-blog.git #一个裸露的仓库
-hooks
-post-receive

3.添加完毕后修改post-receive权限
4.测试git仓库是否可用，在本地主机找空白文件夹

如果能拉取空仓库，则说明git仓库搭建成功了

5.建立ssh信任关系

注：此时的 ssh 登录 git 用户不需要密码！否则就有错，请仔细重复步骤 3-5

6.为了安全起见禁用 git 用户的 shell 登录权限，从而只能用 git clone，git push 等登录 #可以不做

修改 /etc/passwd中的权限

关闭权限后，在本地无法打开git的shell

搭建nginx服务器

1.下载并安装nginx
2.配置nginx文件

先测试是否安装成功

本地操作

发布 配置_config.yaml文件

利用 hexo 配置文件进行链接

1.安装发布插件
2.编辑 _config.yml 的 deploy 属性

3.在 package.json 中添加 npm 脚本

4.在本地调试就用 npm start
5.访问服务器ip即可
